# Scalar type aliases
Define custom types as:
type MyVoltage = Real(units = "V")

Define complicated types in Julia (as abstract or composite Julia types) and declare it as `Native` in Dyad.
Here, MediumModel is a struct defined in Julia and declared as `Native` in Dyad.
```dyad
type MediumModel = Native
```


# Function Type Signatures in Dyad

These define type aliases for function signatures - specifying what arguments a function takes and what it returns.

```dyad
type EnthalpyFunction = func(::Pressure, ::Temperature)::SpecificEnthalpy
#                            ↑ positional args            ↑ return type

type Action = func()::(::Real, ::Real)
#                ↑ no args  ↑ returns tuple of two Reals
```
Use case: When you need to pass functions as parameters to components - common in table lookups, interpolation, or custom property calculations:

```dyad
type RealInterpolator = func(::Real)::Real

component InterpolateFromFile
  parameter filename::String
  structural parameter f::RealInterpolator = dataFromFile(filename)  # Function loaded from file
relations
  y = f(u)  # Call the function parameter
end
```

This enables injecting custom Julia functions (interpolators, property lookups, callbacks) into Dyad models while maintaining type safety - the compiler knows what arguments the function expects and what it returns.
